# Blog — blog.farre.se

Jekyll blog deployed to GitHub Pages via GitHub Actions.

## Creating posts

Posts go in `_posts/` with the naming convention:

```
YYYY-MM-DD-title.md
```

Front matter template:

```yaml
---
title: "Post Title"
date: YYYY-MM-DD
tags: [tag1, tag2]
---
```

## Prose style

Posts should be written in a prose style that sounds like me.

## Comments

We use [utterances](https://utteranc.es/) for comments, backed by GitHub Issues with the `comments` label on `farre/blog`. To add comments to a post, include at the end:

```liquid
{% include comments.html %}
```

It is fine to close a comment issue — users can still comment on closed issues.

## Code snippets

Use Jekyll's highlight tag:

```liquid
{% highlight JavaScript %}
function helloWorld() {
  console.log("Hello, World!");
}
{% endhighlight %}
```

## Tags

Posts are tagged via front matter. Individual tag pages are auto-generated by `_plugins/tag_generator.rb` at `/tags/<tagname>/`. The tag index lives at `/tags/`.

## Building locally

```sh
bundle install
bundle exec jekyll serve
```

## Commits

Do not include "Co-Authored-By" Claude lines in commit messages.

## Deployment

Pushes to `main` trigger the GitHub Actions workflow (`.github/workflows/jekyll.yml`) which builds and deploys to GitHub Pages. The custom domain is `blog.farre.se` (see `CNAME`).
